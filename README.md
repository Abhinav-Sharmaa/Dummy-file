# site-template-generator

Turn any website URL into a reusable **Bootstrap 5 + PHP** look-alike template
(Home + one secondary page) — original code and dummy content only, driven by
measured design tokens and generated by **GitHub Copilot agent mode**.

```
capture.js ──▶ capture/ (screenshots + tokens.json)
                  │
                  ▼
VS Code Copilot Chat ──▶ /generate-template ──▶ template/ (PHP + Bootstrap 5)
                  │
                  ▼
validate.js ──▶ validation/report.html (fidelity scores + deltas to feed back)
```

## Prerequisites

- Node.js ≥ 18
- PHP CLI on PATH (`php -v`) — used by `validate.js` and for previewing
- VS Code with GitHub Copilot (agent mode / Copilot Chat)

## Setup

```bash
npm install
npx playwright install chromium
```

## Usage loop

**1. Capture**

```bash
npm run capture -- https://example.com
# explicit secondary page + font mapping:
npm run capture -- https://example.com /about --font=auto
```

Flags:
- `--font=open-sans` (default) — Open Sans everywhere, measured sizes/weights kept
- `--font=auto` — map each captured family to the closest free Google Font
- `--out=capture` — output directory

Auto-picks the secondary page from nav links (about > services > contact >
team > work > pricing). Writes `capture/tokens.json` + 6 PNGs.

**2. Generate** (Copilot)

Open this folder in VS Code → Copilot Chat → **Agent mode** → type
`/generate-template` and **attach the PNGs from `capture/`** (Add Context →
Files). The prompt file at `.github/prompts/generate-template.prompt.md`
carries the full spec. Output lands in `template/`.

**3. Validate**

```bash
npm run validate                  # default threshold 95
npm run validate -- --threshold=90 --secondary-file=about.php
```

Serves `template/` via `php -S`, re-captures at 375/768/1440, and scores
section geometry (60%) + downscaled layout similarity (40%). Open
`validation/report.html` for side-by-side pairs. Exit code 0 = pass.

**4. Iterate**

Paste the per-section deltas from the console (or `validation/report.json`)
into the same Copilot chat, let the agent fix, re-run `npm run validate`.
Repeat until ≥95 or plateau.

**Preview**

```bash
php -S localhost:8000 -t template
```

## What the capture stores (and doesn't)

`tokens.json` contains measured values only: colors, font metrics, spacing,
section geometry, image dimensions/ratios, animation timing + numeric
keyframes, hover-state deltas, and text **length** stats. It never stores
source text (short nav labels excepted), source CSS/JS, images, or asset
URLs — and the prompt file forbids the agent from reproducing any of them.
Don't reuse source-site assets or branding in generated templates.

## Limitations

- SPAs, auth walls, and heavy anti-bot sites may capture partially.
- Cross-origin stylesheets can't be read → breakpoint list may be partial
  (layout still validated via screenshots).
- Animations are best-effort: only what the Web Animations API exposes at
  load time; scroll-triggered timelines may be approximated.
- Canvas/WebGL/video sections become static fallbacks marked
  `data-fallback` — review these manually.
- Expect 85–95% out of the gate; the validate → paste-deltas loop closes
  most of the rest.

## Troubleshooting

- **`php` not found** — install PHP CLI, ensure it's on PATH, or preview with
  any static-capable PHP host.
- **Port busy** — `npm run validate -- --port=8091`.
- **Playwright browser missing** — `npx playwright install chromium`.
- **Wrong secondary page chosen** — pass the path explicitly:
  `npm run capture -- https://example.com /services`.
